### 4️⃣ **来料不良索赔全流程系统（HS项目）** (2025.10-11)

**项目规模：** 3人团队 | **周期：** 2个月 | **所在公司：** 歌尔智能

**业务背景：**

*   来料索赔依赖线下沟通、纸质审批，流程断点多
*   索赔周期长、损失回收率低、供应商绩效难以量化

**核心成就：**

*   ✅ 实现全流程线上化闭环管理（发现→FA分析→责任判定→成本评估→索赔核算→采购求偿→财务闭环）
*   ✅ 索赔周期平均缩短 **35%**，漏算率下降 **70%+**
*   ✅ 各部门任务流转自动化，审批透明可追踪
*   ✅ 建立供应商质量与索赔绩效数据库

**技术架构：**

*   后端：.NET Core + WebAPI + SqlSugar ORM + DDD
*   前端：Vue 3 + Vol.Web + Ant Design Pro
*   数据库：MySQL
*   工作流：Sys\_WorkFlow（动态配置审批流）
 

**关键难点解决：**

| 难点       | 解决方案                               |
| -------- | ---------------------------------- |
| 复杂多环节审批流 | 状态机双向映射（Status ↔ StepId），支持并行节点与驳回 |
| 索赔金额核算复杂 | DDD模型设计，聚合根+多实体自动计算                |
| 跨系统数据打通  | 统一DataSync API层，自动反写               |
| 供应商沟通追溯  | 多附件上传，自动生成电子档案                     |
| 可视化分析    | 实时看板：金额趋势、TOP10供应商、不良分布            |

### 5️⃣ **HWTE项目 - 设备异常监控与处理系统** (2025.11-12)

**项目规模：** 3人团队 | **周期：** 2个月 | **所在公司：** 歌尔智能科技有限公司

**业务背景：**
生产线设备异常依赖人工被动发现，缺乏实时监控和主动预警机制。需要建立数据驱动的、主动的、可追溯的、持续优化的智能运维体系。

**核心需求：**

1.  **设备异常监控** - 实时侦测设备异常并及时报警
    *   规则1：工位使用率过低（连续X分钟无产品投入）
    *   规则2：工站出现不良（连续X次Fail 或 X小时内Y次Fail）
    *   需排除非工作时间（休息、吃饭、班次交接、节假日）

2.  **异常通知** - 自动分配责任人并支持多级升级
    *   0分钟：通知当站DRI
    *   10分钟未处理：通知DRI Leader
    *   20分钟未处理：通知部门负责人
    *   灵活配置升级时间

3.  **异常跟踪** - 记录处理状态与处理进度
    *   状态流转：待处理 → 处理中 → 已解决
    *   关闭时填写：异常类型、原因、处理措施
    *   下拉选项由管理员维护

4.  **数据分析** - 统计异常发生频率、类型、影响范围（暂缓）

**核心成就：**

*   ✅ 完成生产线层级管理（项目/线体/工站/工位）
*   ✅ 实现工程日历管理（班次、休息、节假日、循环模式）
*   ✅ 设计监控规则配置（规则1、规则2、升级级别）
*   ✅ 实现责任人链管理（多级责任人绑定）
*   ✅ 完成MES数据接入（过站数据、不良数据增量获取）
*   ✅ 设计规则引擎核心（基于工程日历的有效工作时间计算）
*   ✅ 防重复报警机制（异常闭环后才进行下次报警）
*   ✅ 总体业务提效：**每天3小时**

**技术架构：**

*   后端：.NET Core 6 MVC + SqlSugar ORM + DDD领域建模
*   前端：Layui + ECharts 图表库

**关键设计决策：**

| 设计点  | 决策            | 理由          |
| ---- | ------------- | ----------- |
| 配置粒度 | 每个工位独立配置      | 不需继承，灵活性更高  |

| 报警机制 | 闭环后才触发下次      | 防止重复报警      |
| 数据存储 | Redis + MySQL | 实时性与持久化结合   |
| 规则扩展 | 策略模式          | 便于新增规则类型    |
| 通知升级 | 时间驱动          | 自动升级，无需人工干预 |

**核心算法设计：**

**规则1 - 无产出超时检测：**

    1. 获取工位最后过站时间（in_station_time）
    2. 获取当前时间
    3. 计算时间差 = 当前时间 - in_station_time
    4. 获取工位绑定的工程日历
    5. 过滤非工作时间（休息、吃饭、班次交接、节假日）
    6. 计算有效工作时间差
    7. 判断是否超过规则配置的时间 → 触发报警
    8. 检查是否存在未闭环事件 → 防重复报警

**规则2 - 连续不良检测：**

    1. 获取工位在时间窗口内的所有不良记录
    2. 按测试项分组统计（ERROR_CODE分号分隔取第一项）
    3. 获取工位绑定的工程日历
    4. 过滤非工作时间的不良记录
    5. 判断是否满足规则条件：
       - 连续X次Fail（同一测试项）
       - X小时内Y次Fail（同一测试项）
    6. 触发报警
    7. 检查是否存在未闭环事件 → 防重复报警

**数据模型：**

核心实体（16张表）：

*   生产线层级：Project, Line, Station, WorkPosition
*   工程日历：EngineeringCalendar, CalendarEntry
*   监控规则：InactivityRule, DefectRule, EscalationLevel
*   责任人：ResponsibleChain, ResponsiblePerson
*   异常处理：AlertEvent, AlertWorkOrder, HandlingRecord, ExceptionDetail
*   通知：Notification
*   基础数据：BaseDataDictionary

**Redis数据结构：**

*   `station:lasttime:{STATION_CODE}` → 工位最后过站时间
*   `station:daily:{YYYY-MM-DD}` → 当日过站记录集合
*   `defect:count:{STATION_CODE}:{YYYY-MM-DD}` → 不良统计
*   `defect:records:{STATION_CODE}:{YYYY-MM-DD}` → 不良记录时间序列
*   `defect:allrecords:{STATION_CODE}:{YYYY-MM-DD}` → 所有记录（用于追溯）

## 6️⃣ **BOM数据管理系统** (2025.12-2026.01)

**项目规模：** 2-3人团队 | **周期：** 2个月 | **所在公司：** 歌尔智能科技有限公司

**业务背景：**
NPI阶段Config非常多，客户Build Matrix变化频繁，目前BOM核对都是靠人工，工作量巨大且有较大的漏失风险。需要建立统一的BOM数据管理平台，支持多种BOM格式的导入、解析、对比和转换。

**核心需求：**

**模块1：HSD BOM数据管理**

*   导入FATP完整BOM，自动解析颜色信息
*   生成树形结构，支持PC/SC组件分离
*   基础BOM构建（0/1/2阶物料管理）
*   最终BOM生成（CONFIG配置管理）
*   BOM版本控制与变更历史
*   BOM差异对比（拖拽比对、差异识别、物料替换）
*   PLM集成（CFG BOM生成、字段映射、GPN匹配）

**模块2：SMT BOM数据管理**

*   支持多种BOM格式导入（Agile BOM、Build Matrix、GTK Agile BOM、客户CBOM）
*   自定义字段对比（支持勾选对比字段）
*   多场景BOM对比：
    *   Agile BOM vs Build Matrix
    *   Agile BOM vs Agile BOM（版本对比）
    *   Build Matrix vs Build Matrix（CFG对比）
    *   Matrix BOM vs GTK Agile BOM
    *   Agile BOM vs GTK Agile BOM
    *   客户CBOM vs GTK BOM
*   高亮显示单个位号差异
*   GPN重复处理机制（合并位号、替代料检查）
*   两层BOM合并一层BOM（TOP/BOT合并）
*   PLM校验（与PLM系统BOM核对）

**核心成就：**

*   ✅ 统一的BOM数据管理平台完成上线
*   ✅ 支持多种BOM格式的自动解析与转换
*   ✅ 灵活的自定义对比功能，支持6种对比场景
*   ✅ 完整的版本控制与变更追踪体系
*   ✅ 与PLM系统无缝集成，自动校验
*   ✅ 业务提效：BOM核对时间↓80%，漏失风险↓95%

**技术架构：**

*   后端：.NET Core 6 MVC + SqlSugar ORM + DDD领域建模
